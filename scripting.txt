📌 🧠 MOST ASKED INTERVIEW QUESTIONS
Be ready to answer:

Difference: reset vs revert
Both commands "undo" changes, but they do so in fundamentally different ways.
git reset: Moves the current branch pointer backward to a previous commit. It effectively "erases" history from the branch.
Best for: Local changes that haven't been shared with others.

git revert: Creates a new commit that does the exact opposite of the targeted commit. It doesn't erase history; it adds to it.
Best for: Public branches, as it doesn't rewrite history and is safer for collaborators


Difference: merge vs rebase
git merge: Combines the histories of two branches. It creates a "merge commit" that has two parent commits. It is non-destructive and preserves the chronological order of how things happened.
git rebase: Moves or "slides" the entire sequence of commits from one branch onto the tip of another. It rewrites history to create a linear path.
Pro-tip: Never rebase branches that have been pushed to a public repository.

What is staging area?
The Staging Area (or "Index") is a middle ground between your working directory and your repository.
When you use git add, you are telling Git, "I want these specific changes to be part of my next snapshot." This allows you to craft precise commits even if you've modified ten different files but only want to save changes for two of them right now.

What is conflict? how to resolve?
A Merge Conflict occurs when Git cannot automatically determine which changes to keep—usually because two people modified the same line in the same file, or one person deleted a file that another person was editing.

How to resolve:
Locate: Git will mark the conflict in the file with arrows: <<<<<<< HEAD, =======, and >>>>>>>.
Edit: Manually choose which code to keep (or combine them) and remove the Git markers.
Stage: Run git add <file> to mark it as resolved.
Commit: Finalize the merge with git commit.

fetch vs pull
git fetch:  
git pull: Does a git fetch followed immediately by a git merge. It updates your local code with the remote version.

What happens in git commit?
When you run git commit, Git takes a snapshot of everything currently in the staging area.

Creates a Tree Object: It maps the folder structure and files.
Generates a SHA-1 Hash: A unique ID (e.g., f7a2b3...) is generated based on the content and metadata (author, date, message).
Updates the Pointer: The HEAD pointer moves forward to this new commit, making it the latest version of your current branch.
